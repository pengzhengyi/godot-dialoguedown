# Security policy

## Supported versions

DialogueDown is in early development and hasn't published stable releases
yet. Security fixes apply to the latest commit on the default branch unless a
release policy is added later.

## Reporting a vulnerability

Please don't report security vulnerabilities in public issues. Use GitHub's
private vulnerability reporting for this repository instead.

Open the repository's **Security** tab, choose **Report a vulnerability**, and
submit the private report. GitHub will notify maintainers without making the
details public.

When reporting a vulnerability, include:

- Affected version or commit.
- Reproduction steps.
- Expected and actual behavior.
- Potential impact.
- Any suggested fix or mitigation.

Maintainers should acknowledge valid reports, investigate promptly, and publish
a fix or mitigation when appropriate.
