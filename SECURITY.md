# Security policy

## Supported versions

DialogueSystem is in early development and hasn't published stable releases
yet. Security fixes apply to the latest commit on the default branch unless a
release policy is added later.

## Reporting a vulnerability

Please don't report security vulnerabilities in public issues.

> [!WARNING]
> Add a private security contact or enable GitHub private vulnerability reporting
> before the first public release.

When reporting a vulnerability, include:

- Affected version or commit.
- Reproduction steps.
- Expected and actual behavior.
- Potential impact.
- Any suggested fix or mitigation.

Maintainers should acknowledge valid reports, investigate promptly, and publish
a fix or mitigation when appropriate.
